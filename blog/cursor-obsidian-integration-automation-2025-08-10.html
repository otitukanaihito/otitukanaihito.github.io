Cursor×Obsidian連携で日次ワークを短縮する実践ログ

要約 / Summary

Cursor×Obsidian連携と自動化により、日次ワークの作成〜公開までの所要時間を短縮。バックアップ/アーカイブを除外した日付修正、統合ランチャーにCursor起動（メニュー10）を追加し、再現性のある運用を確立しました。

(English) Using Cursor×Obsidian integration and automation, we reduced end-to-end daily workflow time. Safe date correction with backup/archive exclusion and a unified launcher (menu 10) enable a repeatable operation.

今日の実装サマリー

  • 日付修正ツールに除外ルール追加（バックアップ/アーカイブ/ログ）
  • 誤記5件を安全に修正（自動バックアップ＋レポート出力）
  • 統合ランチャーに「Cursor連携（10）」を実装
  • 空ファイル10件を削除、重複の把握のみ（バックアップは保持）

ワークフロー設計

  1. Obsidianで下書き → ランチャー起動（10）でCursorに送る
  2. Cursorで整形/編集 → 自動スクリプトで日付/テンプレ反映
  3. 公開前チェック（リンク・OG・KPI）→ GitHub Pages更新

KPI / 計測

  • 記事作成〜公開のリードタイム: -10分を目標
  • 自動修正件数/空ファイル削減/重複グループ推移のログ化
  • GA: ページビュー・内部遷移・CTAクリックをトラッキング

再現手順（3ステップ）

  1. 自動化スクリプト/open_in_cursor.bat の Cursor.exe パスを確認
  2. ランチャー（10）でインデックスをCursorで開き、記事テンプレを作成
  3. 日付修正プレビュー → 実行 → 公開チェック（OG/リンク/見出し）

つまずきと回避策

  • PowerShellのエンコード/パイプ問題 → UTF-8強制とパイプ削除で解消
  • バックアップを誤修正しない → ディレクトリ除外の明示（prefix含む）
