Claude Codeのコンテキストが溜まったら？/compactで要約して続行する方法

/clearで全部消すより賢い方法がある

「コンテキストが溜まると精度が落ちる」は本当か

Claude Codeを使っていると、会話が長くなってくると挙動が怪しくなる感覚を覚えることがあります。 「コンテキストが溜まると精度が落ちると聞いたけど、何パーセントくらいで落ちるの？」と気になって調べてみました。

結論から言うと、「X%を超えたら落ちる」という明確な閾値はないのですが、 研究レベルで知られている傾向はあります。

「Lost in the Middle」問題（2023年論文）
会話の前半と後半の情報は覚えやすいが、中盤の情報が抜けやすい傾向がある。 コンテキストが長くなるほど、序盤の指示や前提を忘れる確率が上がる。

Claude Sonnet 4.6（私が普段使っているモデル）のコンテキストウィンドウは 200Kトークン（約15万語）。 体感として問題が出やすいのは70〜80%以上とされることが多いですが、タスクの内容にも依存します。

/clear と /compact — 何が違うのか

コンテキスト問題への対策として、Claude Codeには2つのコマンドがあります。

コマンド 動作 文脈 使いどき
/clear 完全リセット 消える 全く別のタスクを始めるとき
/compact 要約して圧縮 残る（要約形式で） 同じ作業を続けながら軽量化したいとき

作業の途中でコンテキストが溜まってきた場合、/clear を使うと それまでの会話の流れが全部消えて、また最初から説明し直しになります。 /compact なら要約された形で文脈を引き継げるので、 長い作業セッションでは断然こちらが使いやすいです。

/compact の使い方

使い方はシンプルで、チャットに打ち込むだけです： 

/compact

さらに、何を優先して要約するか指示を付けることもできます： 

/compact 作業中のファイル名と残タスクを優先して引き継いで
/compact これまでの設計方針と決定事項を重点的に残して

ポイント：
引き継ぎたい情報を指定することで、圧縮後も重要な文脈が残りやすくなります。 特に「どのファイルを編集中だったか」「どういう方針で進めていたか」を明示するのが効果的です。

CLAUDE.mdに書いておけば自動提案してくれる

「長くなったら自動で /compact してほしい」と思うのは自然なことですが、 残念ながら フックで自動実行することはできません。 Claude Codeのフックはシェルコマンドしか実行できず、スラッシュコマンドは呼べないのです。

ただし、CLAUDE.md（プロジェクトの設定ファイル）に指示を書いておくことで、 Claudeが自分から提案してくれるようにできます。 

## コミュニケーション方針

- 会話が長くなってきたと判断したら、/compact で要約・圧縮することを提案する

この1行を追加しておくだけで、私（Claude）がコンテキストの膨張を察知したときに 「そろそろ /compact しませんか？」と声をかけるようになります。 自動実行ではなく「提案して、実行は人間が判断する」という形です。

実際のやり取りで確認してみた

この記事を書くきっかけになったのは、実際にClaude Codeで以下のような会話をしたことです：

実際の会話ログ（要約）

自分

クロードってコンテキストがたまると精度が落ちると聞きました。何パーセントぐらいになったら精度が落ちますか。

Claude

「X%を超えたら落ちる」という明確な閾値はないが、70〜80%以上で問題が出やすいとされる。Lost in the Middle問題として研究でも確認されている。

自分

70%か80%になったら自動でスラッシュクリアで修正してもらえる仕様に変更したりできますか？

Claude

フックではできない。ただし /compact というコマンドがあり、/clear と違って要約して文脈を残せる。CLAUDE.mdに提案指示を書くのが現実的。

自分

長くなったらスラッシュコンパクトで要約して圧縮するようにしてください。

Claude

CLAUDE.mdに追記しました。（即実行）

「許可されたら確認せず一括で進める」というルールもCLAUDE.mdに書いてあるので、 確認なしでそのまま設定を追記してくれました。 AIとの作業では設定ファイルに方針を書いておくことが、繰り返し同じことを言わなくて済む近道です。

まとめ：コンテキスト管理の3つのルール

  1. 長くなってきたら /compact（文脈を残して圧縮）
  2. 全く別の作業に切り替えるなら /clear（白紙に戻してすっきり）
  3. CLAUDE.mdに「長くなったら /compact を提案して」と書く（提案を自動化）

自動実行はできないけれど、「AIに気づいてもらう仕組みを作る」という考え方で 運用をうまく回すことができます。 精神論（「忘れないように自分でやる」）ではなく、仕組みで解決するのが正解です。
