Claude Codeスキルの探し方・インストール方法
〜自分に合ったスキルをネットから入れる〜

SKILL.md 1つで「専門家を雇う」感覚。34個使ってわかった活用のコツ

はじめに

Claude Codeを使い始めてしばらくすると、「スキル」という機能に出会います。 私は最初「なんか難しそう」と思って後回しにしていたのですが、 使い始めたら 「もっと早く使えばよかった」 と思うくらい便利でした。

福祉のデイケアで働きながら副業ブログをやっている私にとって、 時間は本当に貴重です。「ブログ記事書いて」の一言でAIチームが動く、 「歌詞をPDFにして」で利用者さん向けの印刷物が完成する。 そんな仕組みをスキルで実現しています。

この記事では、Claude Codeのスキルとは何か、どこで探して、どうやってインストールするかを 実体験ベースでわかりやすく解説します。

この記事の対象読者：
Claude Code を使っているが、スキル機能をまだ活用していない方。 「スキルって何？」から「実際にどうインストールするの？」まで一通り知りたい方。

1. Claude Codeの「スキル」って何？

一言で言うと、Claude Codeに特定の専門知識や手順を教えるMarkdownファイルです。

ファイル名は SKILL.md。これを決まったフォルダに置くだけで、 Claude Codeが会話中に自動で参照して、適切な場面で実行してくれます。

イメージとしては「専門家を雇う」感覚に近いです。 SEOのことならSEOスキル、ブログ記事なら記事作成スキル、 Gitの操作ならgit-workflowスキルが動く。 毎回同じことをAIに説明しなくて済むようになります。

スキルのフォルダ構成（グローバルの場合）

~/.claude/skills/
├── blog/
│   └── SKILL.md      ← ブログ記事作成スキル
├── seo/
│   └── SKILL.md      ← SEO分析スキル
├── git-workflow/
│   └── SKILL.md      ← Git操作スキル
└── lyrics-print/
    └── SKILL.md      ← 歌詞PDF化スキル（職場用）

スキルには2種類の置き場所があります：

  • グローバルスキル：~/.claude/skills/ — どのプロジェクトでも使える
  • プロジェクトローカルスキル：.agent/skills/（プロジェクト内） — そのフォルダ専用

SKILL.md の中身はシンプル

難しそうに見えて、中身はMarkdownのテキストファイルです。 「このスキルが呼ばれたら、こういう手順でこういう出力をしてください」と書くだけ。

SKILL.md の最小構成例

---
name: weekly-review
description: "週次振り返りを作成する。「週次振り返り」「週の振り返り」などで発動"
---

# 週次振り返りスキル

## 概要
今週の作業ログをもとにKPT形式の振り返りを作成する。

## 手順
1. 作業ログフォルダの最新ファイルを読み込む
2. Keep / Problem / Try に分類する
3. 来週のアクション3件を提案する

2. スキルはどこで探す？4つの方法

方法1：find-skillsスキルを使う（最も簡単）

「find-skills」というメタスキルがあります。 Claude Codeに「こんなスキルない？○○ができるやつ」と聞くだけで、 インターネット上から自動的に探してきてくれます。

私はこれを最初にインストールしました。 「デプロイを自動化できるスキルない？」と聞いたら、 site-deployスキルを見つけてインストール方法まで教えてくれました。

方法2：awesome-claude-skillsで探す

GitHub上にコミュニティが厳選したスキル一覧があります： awesome-claude-skills

「awesome-〇〇」シリーズはGitHubではおなじみの厳選リスト形式で、 品質が高いスキルが集まっています。初めて探すならここから見るのがおすすめです。

方法3：スキルマーケットプレイスで探す

サービス URL 特徴
SkillsMP skillsmp.com スキルのマーケットプレイス。カテゴリ検索が使いやすい
SkillHub skillhub.club スキル共有プラットフォーム。コミュニティ評価つき
Anthropic公式 github.com/anthropics/skills Anthropic公式提供スキル。信頼性が高い

方法4：GitHub検索

GitHubの検索で Claude Code skill SKILL.md [キーワード] と入力すると、 個人開発者が公開しているスキルが出てきます。 ニッチな用途のスキルはこちらで見つかることも多いです。

3. インストールは3ステップ（mkdir → curl → 完了）

スキルのインストールは拍子抜けするほど簡単です。 コマンド3行で終わります。

スキルのインストール手順（共通）

# 1. スキル用フォルダを作成
mkdir -p ~/.claude/skills/[スキル名]

# 2. SKILL.mdをダウンロード
curl -sL [GitHubのraw URL] > ~/.claude/skills/[スキル名]/SKILL.md

# 3. 確認
ls ~/.claude/skills/[スキル名]/
# → SKILL.md が表示されればOK

Claude Codeを再起動（または新しいセッションを開始）すると、 自動でスキルが認識されます。設定ファイルへの追記などは不要です。

インストール後の確認方法：
Claude Codeで「インストールしたスキルの名前」や「そのスキルが発動するキーワード」を含む指示を出してみてください。 スキルが認識されていれば、対応した動作をしてくれます。

Windowsユーザーへの補足

Windowsで使う場合、~/.claude/skills/ は C:\Users\[ユーザー名]\.claude\skills\ に対応します。 mkdir コマンドはGit BashやWSL環境で実行するとスムーズです。

4. 私が入れている34個のスキル（カテゴリ別）

現在、私の環境には34個のスキルが入っています。 カテゴリ別に紹介します。

チーム管理系スキル（5個）

複数のAIエージェントを連携させるための「司令塔」系スキルです。 「ブログ記事書いて」と指示すると、team-managerスキルが発動して リサーチ・記事作成・レビューを分担してくれます。

  • team-manager：マルチエージェントの司令塔
  • agent-content：ブログ記事作成担当
  • agent-research：情報収集・調査担当
  • agent-system：システム設計・実装担当
  • agent-video：動画制作担当

ブログ・コンテンツ系スキル（2個）

  • blog：記事作成の全工程エンジン（v1.6.0）。タイトルからHTML出力まで
  • copywriting：マーケティングコピー作成

SEO系スキル（13個）

SEO関連は専門性が高いため、用途ごとに細かく分かれています。 「SEOチェックして」と言うだけで、seoスキルが関連サブスキルを呼び出してくれます。

ユーティリティ系スキル（7個）

  • find-skills：スキル検索メタスキル（前述）
  • skill-creator：新スキルを作るためのテンプレートを提供
  • git-workflow：Git操作・PR管理
  • site-deploy：GitHub Pagesへのデプロイ手順
  • google-ai-mode-skill：Google AI検索で最新情報を取得
  • obsidian-cli：Obsidianノートの操作
  • weekly-review：週次振り返りの作成

福祉・業務系スキル（2個）

デイケアの仕事に使うスキルです。職場では外部ツール不可ですが、 個人PCでの準備作業にスキルを活用しています。

  • lyrics-print：歌詞テキストをA4印刷用HTMLに変換（デイケア利用者向け）
  • voice-memo-organizer：音声メモをナレッジベースに整理・保存

5. スキルを選ぶ時の注意点

インストール前に中身を確認すること：
スキルはMarkdownテキストなので、インストール前に必ずSKILL.mdの中身を読んでください。 信頼できる作者・GitHubスター数が多い・最終更新日が新しい、 これらを確認してから入れるのが安全です。

スキルを選ぶときの評価基準はこの3点です：

  1. GitHubスター数・更新日：メンテナンスが続いているか確認する
  2. SKILL.mdの品質：name/descriptionが明確で、手順が具体的か
  3. 自分の用途に合うか：descriptionに書かれたトリガー条件が自分の使い方と一致するか

また、同名スキルが ~/.claude/skills/ と .agent/skills/ の両方にある場合、 両方が読み込まれます。意図しない動作を防ぐため、 同名スキルの二重インストールには注意してください。

6. 自分でスキルを作る方法

既存のスキルに合うものがなければ、自分で作ることもできます。 skill-creatorスキルを使うと、テンプレートを提供してくれます。

私の「lyrics-print」スキルは完全に自作です。 「デイケアの利用者さん向けに歌詞をA4印刷用のHTMLに変換する」という ニッチすぎる用途はネットにないので、自分で作りました。

スキル作りで大事なのはdescriptionの書き方です。 Claude Codeはここを読んで「このスキルを使うべきか」を判断するので、 どんな言葉で話しかけたときに発動してほしいかを具体的に書くのがコツです。

descriptionの書き方のコツ

---
name: lyrics-print
description: "歌詞テキストをA4印刷用HTMLに変換する。
  「歌詞をPDFにして」「歌詞を印刷用に」「歌詞.txtを変換して」
  などで発動。デイケア利用者向けのレイアウト。"
---

7. まとめ：スキルでClaude Codeが「自分専用のAIチーム」になる

この記事のポイントまとめ

  • 1. スキル = SKILL.mdファイル。難しいものではなく、Markdownで書ける指示書
  • 2. 探し場所は「find-skillsスキル」「awesome-claude-skills」「SkillsMP」「GitHub検索」の4つ
  • 3. インストールは mkdir → curl → 再起動の3ステップだけ
  • 4. インストール前に中身（SKILL.md）を必ず確認する
  • 5. ニッチな用途は自作する。skill-creatorスキルがテンプレートを出してくれる

スキルを増やしていくと、Claude Codeが「自分の仕事を理解している専任チーム」に育っていく感覚があります。

ブログ担当、SEO担当、調査担当、職場の歌詞担当……それぞれの専門家が、 適切な言葉をかけると動き出す。そんな体制が、 Markdownファイルを数個並べるだけで作れてしまうのがスキルの面白さです。

まずは1つ、自分の仕事で「毎回同じことをAIに説明してるな」という作業を探してみてください。 そこをSKILL.mdにすることから始めると、スキルの価値が実感できます。
