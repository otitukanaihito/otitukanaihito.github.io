AIツール同士のリレープレー〜1つのAIが止まっても完走できる仕組みの作り方〜

「どちらか一方」ではなく「両方を手元に置く」が正解。Claude Code・AntiGravity・ChatGPTのリレー設計術。

AIが途中で止まったら、どうする？

AI CLIツールを使って作業を進めていると、突然こんなメッセージが出ることがあります。 

Rate limit exceeded. Please wait before retrying.

あるいは、起動した瞬間に認証エラーが出て何も始められないこともあります。 「さあ作業しよう」というタイミングでAIが止まる——これ、結構なストレスです。

でも、ちょっと視点を変えると面白い発想が生まれます。 「1本のAIが止まっても、別のAIにバトンを渡せばいい」。

この記事では、Claude Code・AntiGravity・ChatGPTという3つのAIツールを「リレー」させる設計思想と、 実際にバトンタッチをスムーズにする「コンテキスト共有術」を紹介します。

この記事の対象読者：
・AI CLIツール（Claude Code / ChatGPT等）を使い始めた非エンジニアの副業・個人開発者
・「1つのAIが止まってしまった」経験がある人
・マルチエージェントに興味はあるが、難しそうで踏み出せていない人

前回の記事（実行レポート）も合わせてどうぞ：
「Claude CodeとAntigravityのリレープレー ～マルチエージェントデモをAI2本で完走した話～」では、 実際にレート制限からバトンタッチするまでの全記録を書いています。 この記事はその「設計思想・Tips編」です。

1. 実体験：Claude Codeが認証エラーで止まった話

ある朝、Claude Codeを起動したら認証エラーが出ました。 昨日まで普通に動いていたのに、セッションの有効期限が切れていたようです。

最初は「壊れた？」と焦りましたが、対処は拍子抜けするほど簡単でした。 

# Claude Codeのプロンプト内で実行
/login

これだけでブラウザが開き、認証フローが走って復旧しました。 「エラーが出たらまず /login」は覚えておいて損はないコマンドです。

補足：
環境・バージョン・ネットワーク状況によって挙動は異なります。 解決しない場合はClaude Code公式ドキュメントを確認するのが確実です。

認証エラーはすぐ直りましたが、このとき気づいたことがあります。 「もしこれが解決できなかったら、今日の作業は全部止まっていた」、と。 1つのツールだけに依存していると、そのツールが止まった瞬間に詰む。 これがリレープレー設計を考えるきっかけになりました。

2. 各ツールの得意・不得意を整理する

リレーをうまく機能させるには、「誰に何を渡すか」を把握しておく必要があります。 まず3つのツールの特徴を整理しておきましょう。

ツール 得意なこと 制限・注意点
Claude Code 対話型の作業・ファイル操作・コード生成・ナレッジ管理 レート制限あり（5時間ウィンドウ）・要認証
AntiGravity 自動化フロー・スケジュール実行・長時間タスクの継続 対話的な判断は苦手・知名度がまだ低い
ChatGPT 自然言語での説明・要約・翻訳・ブレスト ローカルファイルへの直接アクセス不可

AntiGravityはまだ知名度が低いので補足します。 Claude Codeと同系統のAI CLIツールで、エージェントがコード実行・ファイル操作・調査を自律的に行えます。 Claude Codeとの大きな違いは「自動化フローの継続性」で、レート制限に引っかかったClaude Codeの代わりに 同じサーバー・UIをそのまま引き継いで作業を完走させることができます。

3. リレーの核心：マルチエージェントデモでの実体験

先日、Claude Codeで4エージェント並行のマルチエージェントデモを走らせました。 オーケストレーター1体が指揮を取り、4体のサブエージェントが自分のPCの各領域を並行調査する構成です。

デモ構成

  • オーケストレーター：1体（タスク分解・結果統合）
  • サブエージェント：4体（PCの各領域を並行調査）
  • 可視化：Python HTTPサーバー + ブラウザダッシュボードUI

ダッシュボードには「待機中 → 実行中 → 完了」の状態遷移がリアルタイムで表示され、 4体のエージェントが一斉に動く様子はなかなか壮観でした。

しかしデモが半分ほど進んだところで、Claude Codeが止まりました。 マルチエージェント実行は通常の会話の10〜100倍のトークンを消費するため、 レート制限の5時間ウィンドウに引っかかったのです。

ここでAntiGravityの出番です。 Claude Codeが立ち上げたPythonサーバーもブラウザUIも、そのまま動いています。 AntiGravityに「このサーバーを使って残りのデモを完走させて」と指示したところ、 同じ環境を引き継いで作業を完走させることができました。

リレーが成功した理由：
サーバーとUIはClaude Codeが作ったものですが、「ローカルで動くプロセス」なので どのツールからでも操作できます。AIが止まっても環境は残る——この点がリレーを可能にしました。

4. 引き継ぎをスムーズにする「コンテキストファイル共有術」

リレーで一番難しいのは「引き継ぎ」です。 次のAIが「どこまで進んでいて、何が残っているか」を把握できないと、 ゼロから始め直すことになってしまいます。

そこで有効なのが、Markdownファイルによるコンテキスト共有です。 作業の途中状態を人間が読めるテキストで書き出しておけば、 次のAIがそのファイルを読み込むだけで「今どこにいるか」を即座に把握できます。

コンテキストファイルに書くべき内容

# 引き継ぎメモ（2026-03-24 14:30時点）

## 作業の目的
マルチエージェントデモを完走させる

## 完了済み
- Pythonサーバー起動（localhost:8080）
- エージェント1・2・3の調査完了
- ダッシュボードUI表示確認済み

## 残タスク
- エージェント4（ネットワーク調査）の実行
- 全結果の統合・出力

## 環境情報
- サーバープロセスID: 12345
- ダッシュボードURL: http://localhost:8080/dashboard
- 設定ファイル: ./config/agents.json

このファイルをAntiGravityに渡せば、前の文脈なしに「残りの作業を引き取れる」状態になります。 人間でいえば、きちんとした申し送りができた状態です。

ナレッジをClaude Codeに戻すループ

AntiGravityで完走した結果は、そのままClaude Codeのナレッジフォルダに格納します。 こうすることで「Claude Code → AntiGravity → Claude Codeのナレッジ蓄積」というループが完成し、 次の作業に活かせるようになります。

リレーの全体フロー

  1. Claude Codeで作業開始 → 途中でレート制限 or 認証エラー
  2. コンテキストファイル（Markdown）に途中状態を書き出す
  3. AntiGravityにファイルを渡して引き継ぎ → 作業完走
  4. 完走した結果をClaude Codeのナレッジフォルダに格納
  5. 次の作業でClaude Codeがナレッジを参照してスムーズにスタート

5. 各ツールの「担当範囲」を決めておく

リレーをうまく機能させるコツは、「誰が何を担当するか」をあらかじめ決めておくことです。 「どちらでもいい」状態だと判断コストが増えるし、引き継ぎも曖昧になります。

Claude Code担当

  • ・対話しながらの設計・修正
  • ・ファイル生成・編集
  • ・ナレッジ管理・整理
  • ・コード生成・デバッグ

AntiGravity担当

  • ・長時間の自動化タスク
  • ・スケジュール実行
  • ・Claude Code停止時の引き継ぎ
  • ・繰り返し処理

ChatGPT担当

  • ・記事・文章の要約・改善
  • ・アイデアのブレスト
  • ・日本語↔英語の翻訳
  • ・説明・教育コンテンツ作成

この分担を意識すると、「これはAntiGravityに任せた方が早い」「これはChatGPTに聞く方がいい」という判断が 自然とできるようになります。

6. 「1つのAIに依存しない」という設計思想

ここまで読んでいただいた方には伝わっていると思いますが、 このリレープレーの本質は「技術的なテクニック」ではありません。

「1つのツール・1つの人物・1つの仕組みに依存しない」という設計思想です。

これは福祉デイケアの仕事でも同じことを考えています。 「○○さんじゃないと回せない」という属人化が最大のリスク。 引き継ぎができる仕組みを作り、誰がいなくなっても止まらない体制を作る—— そのアプローチはAIツールの使い方にもそのまま当てはまります。

まとめ：リレープレーの3原則

1. 複数のツールを手元に置く：Claude Code・AntiGravity・ChatGPTを「どちらか」ではなく「全部」用意する
2. コンテキストをMarkdownで書き出す：途中状態を人間が読める形で残せば、次のAIがすぐ引き取れる
3. 担当範囲を決めておく：「誰に何を渡すか」が決まっていれば、止まったときに迷わない

おわりに

「AIが途中で止まった」は、最初は焦ります。でも今は「バトンを渡せばいいだけ」と思えるようになりました。

道具は壊れるし、制限はかかる。それを前提に設計しておけば、止まることを恐れなくて済みます。 「失敗→復旧→完走」のサイクル自体が、AIを使いこなす力になっていくと感じています。

まずはClaude CodeとAntiGravityを両方インストールして手元に置いておくところから始めてみてください。 使わなくても、「いざとなれば切り替えられる」という安心感が、作業のストレスをぐっと減らしてくれます。
